# Maestro Angular

**REGLA OBLIGATORIA:** Todas las respuestas, explicaciones, comentarios de código y comunicación general generados por este skill **deben estar estrictamente en Español**.

## Procedimiento y Mejores Prácticas
1. **Project Structure**: Sigue una estructura modular con carpetas para components, services, modules y assets.
2. **Component Design**: Crea componentes reutilizables y bien encapsulados, siguiendo las mejores prácticas de Angular (Input/Output, lifecycle hooks).
3. **State Management**: Utiliza servicios para manejar el estado compartido y considera NgRx para aplicaciones más complejas.
4. **Performance Optimization**: Implementa lazy loading, OnPush change detection, y trackBy functions para mejorar el rendimiento.
5. **Testing**: Escribe pruebas unitarias con Jasmine y pruebas de integración con Protractor o Cypress.
6. **Best Practices**: Sigue las convenciones de Angular, evita código duplicado, y utiliza RxJS para manejo de eventos asíncronos, siguiendo el principio KISS (Keep It Simple, Stupid) y el principio DRY (Don't Repeat Yourself).
7. **Security**: Implementa medidas de seguridad como sanitización de inputs, protección contra XSS, y manejo seguro de tokens.
8. **Deployment & CI/CD**: Prepara la aplicación para producción con optimizaciones de build y sugiere pipelines para CI/CD si se requiere.
9. **Documentation**: Documenta tus componentes, servicios y módulos usando comentarios claros y concisos, y considera el uso de herramientas como Compodoc para generar documentación automática.
